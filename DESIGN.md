## 设计理念

### 产品定位

AttenLink 是一款**专注事实的AI资讯聚合工具**，旨在帮助用户在信息过载的时代，高效获取真实、有价值的资讯。

### 设计原则

| 原则 | 说明 |
|------|------|
| **专注** | 减少干扰，让用户专注于内容本身 |
| **信任** | 通过AI查证建立用户对信息的信任 |
| **高效** | 智能推荐，减少信息筛选成本 |
| **沉浸** | 全屏卡片设计，沉浸式阅读体验 |

### 核心体验

```
发现 → 阅读 → 互动 → 查证 → 再发现
  ↑                              │
  └──────────────────────────────┘
        (个性化推荐循环)
```

## 用户画像

### 主要用户群体

#### 用户 A：信息筛选者
- **特征**：每天浏览大量资讯，但时间有限
- **痛点**：信息过载，真假难辨
- **需求**：快速获取高质量、已验证的新闻

#### 用户 B：深度阅读者
- **特征**：关注特定领域，追求内容深度
- **痛点**：难以找到专业、可靠的信息源
- **需求**：个性化推荐，事实查证

#### 用户 C：效率追求者
- **特征**：工作繁忙，碎片化时间阅读
- **痛点**：传统新闻App操作繁琐
- **需求**：简洁交互，快速浏览

### 用户旅程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  打开   │───▶│  浏览   │───▶│  互动   │───▶│  查看   │───▶│  继续   │
│  应用   │    │  推荐   │    │  卡片   │    │  详情   │    │  浏览   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
    │              │              │              │              │
    ▼              ▼              ▼              ▼              ▼
 启动画面      全屏卡片展示    左滑/右滑/下滑   完整内容阅读   下一张卡片
 <1秒          沉浸式体验     表达偏好        AI查证结果     循环体验
```

## 动效设计

### 卡片滑动动效

```dart
// 滑动时的视觉反馈
- 卡片跟随手指移动
- 背景颜色根据滑动方向渐变
- 图标随滑动距离放大并淡入
- 释放时根据阈值决定回弹或飞出
```

### 卡片切换动效

```
当前卡片飞出后：
1. 下一张卡片从底部滑入（300ms，easeOut）
2. 卡片缩放从 0.95 到 1.0
3. 透明度从 0.8 到 1.0
4. 背景图片淡入切换
```

### AI 查证面板动效

```
查证完成时：
1. 面板从底部滑入（300ms，easeOut）
2. 查证图标缩放弹出（200ms，elastic）
3. 文字逐行淡入（stagger 50ms）
```

### 主题切换动效

```
深色/浅色模式切换：
1. 圆形扩散动画从切换按钮位置开始
2. 颜色平滑过渡（300ms）
3. 所有组件同步更新
```

## 响应式设计

### 断点定义

| 断点 | 宽度 | 适配设备 |
|------|------|----------|
| Compact | < 600dp | 手机 |
| Medium | 600-840dp | 大屏手机/小平板 |
| Expanded | > 840dp | 平板/桌面 |

### 布局适配

**手机（Compact）**：
- 单卡片全屏展示
- 底部导航栏
- 全宽按钮

**平板（Expanded）**：
- 双列卡片布局
- 侧边导航栏
- 卡片固定宽度

## 无障碍设计

### 可访问性要求

- 支持屏幕阅读器
- 颜色对比度符合 WCAG AA 标准
- 支持字体缩放（最大 200%）
- 所有交互元素最小 48dp 点击区域
- 支持键盘导航
